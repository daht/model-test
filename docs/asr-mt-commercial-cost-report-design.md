# ASR + MT 商业成本总报告设计

日期：2026-07-22

## 目标

把现有 ASR 容量报告、MT 容量报告和百万用户业务量测算合并为一份面向商业决策的总报告，同时提供 Markdown 源文件和内容一致的 HTML 阅读版。

报告只覆盖 ASR 与 MT，不包含 TTS。

## 读者与用途

- 面向管理层、产品、商务和技术负责人。
- 回答百万用户规模需要多少张 A10、每月与每年纯 GPU 成本是多少、推荐哪种 ASR 方案，以及现有证据还缺少哪些生产门禁。
- 不把容量测试结果表述为翻译或识别质量结论。

## 业务输入

- A10 成本：2,132.72 元/月，25,592.64 元/年。
- 百万用户月音频量：438,082,143 秒，等效平均持续并发约 169.01 路。
- 百万用户月翻译量：约 2,142,857,143 个源字符。
- 生产容量统一按推荐单卡容量的 70% 规划，再向上取整业务卡数量，并为 ASR 与 MT 分别增加一张 N+1 冗余卡。
- ASR 与 MT 不混部，因为尚未完成同卡显存、算力竞争与故障隔离验证。

## 推荐方案

### ASR

主方案采用 SenseVoice Small：

- 已验证最高完整成功档：88 路；
- 当前有余量建议：80 路；
- 70% 规划容量：56 路/A10；
- 业务卡：`ceil(169.01 / 56) = 4` 张；
- N+1 部署：5 张 A10；
- 纯 GPU 成本：10,663.60 元/月，127,963.20 元/年。

80 路是当前有余量建议，不是已经完成长期稳定性与质量门禁的生产 SLA。

### MT

主方案采用 Hy-MT2-1.8B + vLLM：

- 推荐 admission：48；
- 一小时实测吞吐：5,727.42 源字符/秒；
- 70% 月规划容量：10,391,823,682 源字符/A10；
- 业务卡：1 张；
- N+1 部署：2 张 A10；
- 纯 GPU 成本：4,265.44 元/月，51,185.28 元/年。

### 合计

- ASR：5 张 A10；
- MT：2 张 A10；
- 总计：7 张 A10；
- 纯 GPU 成本：14,929.04 元/月，179,148.48 元/年。

## ASR 替代方案

报告附带两种 Qwen3-ASR 对照，使用相同 70% 规划与分服务 N+1 口径：

| ASR 方案 | ASR 部署 | 加 MT 后总卡数 | 总 GPU 成本/月 | 总 GPU 成本/年 |
| --- | ---: | ---: | ---: | ---: |
| SenseVoice Small，推荐 | 5 | 7 | 14,929.04 元 | 179,148.48 元 |
| Qwen3-ASR-0.6B | 12 | 14 | 29,858.08 元 | 358,296.96 元 |
| Qwen3-ASR-1.7B | 22 | 24 | 51,185.28 元 | 614,223.36 元 |

模型选择不能只依据容量。报告明确要求使用统一标注语料补充 CER/WER、专名、数字、混合语言、标点和端点质量验收。

## 报告结构

1. 执行摘要与推荐方案。
2. 业务量及成本口径。
3. 推荐部署拓扑与服务隔离原则。
4. SenseVoice ASR 容量、服务器与成本。
5. Hy-MT2/vLLM MT 容量、服务器与成本。
6. ASR + MT 合并成本。
7. 三种 ASR 选择对总成本的影响。
8. 市场 API 公开价格参考与自建盈亏平衡说明。
9. 已验证证据、未验证边界和上线门禁。
10. 数据来源与可追溯性。

## 输出文件

- `docs/asr-mt-commercial-cost-report-2026-07-22.md`：报告源文件。
- `docs/asr-mt-commercial-cost-report-2026-07-22.html`：浏览器阅读版。
- 在 `README.md` 增加报告入口。

两种格式必须使用相同数字、结论和证据状态。HTML 采用现有成本报告的卡片与表格风格，支持窄屏和打印。

## 验证

- 使用脚本从业务输入重新计算所有服务器数量、月成本和年成本。
- 将 ASR 容量与现有三模型报告逐项核对。
- 将 MT 容量、延迟、错误率和成本与一小时原始 JSON 及监控归档哈希核对。
- 校验 Markdown 本地链接、HTML 标签结构、关键数值一致性和敏感信息。
- 运行相关 ASR/MT 文档依赖的现有回归测试。

## 边界

- 成本仅包含 GPU，不包含 CPU、系统内存、磁盘、网络、负载均衡、监控、研发和运维。
- 不把官方模型 benchmark 当作本项目质量验收结果。
- 不保存服务地址、API 密钥、请求文本、译文、音频或识别文本。
- 不宣称 ASR 主方案 production-ready；SenseVoice 仍缺长时间 soak、统一质量门禁和多实例故障切换验证。
